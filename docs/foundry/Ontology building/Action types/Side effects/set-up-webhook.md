# [](#set-up-a-webhook)Set up a webhook设置 webhook


## [](#prerequisites)Prerequisites前提条件


1. Complete the [getting started](/docs/foundry/action-types/getting-started/) tutorial to learn about how to set up a basic action.完成入门教程，了解如何设置基本操作。
2. Set up a [webhook](/docs/foundry/data-connection/webhooks-setup/) to create a connection to an external system. You may need to work with a system administrator who has permissions to connect to source systems before creating a webhook.设置 webhook 以创建与外部系统的连接。在创建 webhook 之前，您可能需要与具有连接源系统权限的系统管理员合作。


## [](#steps)Steps步骤


In the below example, suppose that we have a webhook called **Modify ticket priority** which accepts a `priority` string value and sends it to an external system.在下面的示例中，假设我们有一个名为"修改工单优先级"的 webhook，它接受一个 priority 字符串值并将其发送到外部系统。


Navigate to your action, select the **Logic** tab, select **Add new rule**, then select **Webhook**.导航到您的操作，选择逻辑选项卡，选择添加新规则，然后选择 Webhook。


![Add new webhook](webhooks-add-new-rule-webhook.png?width=600)


By default, the newly added webhook is configured as a **side effect**; this makes the webhook run *after* objects are edited in Foundry. Alternatively, select **Writeback** to have the webhook run *before* object edits.默认情况下，新添加的 webhook 配置为副作用；这使 webhook 在 Foundry 中编辑对象后运行。或者，选择写入回传以使 webhook 在对象编辑之前运行。


![Select writeback or side effect](webhooks-select-writeback-or-side-effect.png?width=400)


In the menu below, select the webhook you want to execute. In this example, we select the **Modify ticket priority** webhook. If you wish, you can select a different version of the webhook.在下面的菜单中，选择您要执行的 webhook。在这个示例中，我们选择了"修改工单优先级"webhook。如果您愿意，可以选择 webhook 的不同版本。


![Select webhook to execute](webhooks-select-webhook-to-execute.png?width=400)


Next, configure webhook input parameters based on action parameters. By default, a new action parameter is generated for each webhook input unless an existing action parameter with the same name already exists, in which case the existing parameter is automatically configured. In this example, we map the webhook input called `priority` to an existing action parameter called `Ticket Priority`.接下来，根据操作参数配置 webhook 输入参数。默认情况下，为每个 webhook 输入生成一个新的操作参数，除非已存在同名操作参数，此时将自动配置现有参数。在本例中，我们将名为 priority 的 webhook 输入映射到名为 Ticket Priority 的现有操作参数。


![Mapping webhook input to an existing action parameter](webhooks-input-mapping-example.png?width=400)


Select **Add webhook** to complete adding the webhook rule to your action. Note that after the webhook is added, the Rules section shows that the writeback will occur before objects are modified.选择添加 webhook 以完成将 webhook 规则添加到您的操作。注意，添加 webhook 后，规则部分显示回写将在对象修改之前发生。


![Add new webhook (completed)](webhooks-add-webhook-complete.png?width=600)


You may want to remove any autogenerated parameters created when you added the webhook which are now unused.你可能需要删除在添加 webhook 时自动生成的、现在不再使用的任何参数。


![Remove autogenerated parameters](webhooks-remove-autogenerated-parameters.png?width=600)


Select **Save** in the top-right to save your changes to the action. Now, when the action is applied, the request to the external system will be made before any changes are applied to objects in Foundry.在右上角选择“保存”以保存对操作的更改。现在，当应用操作时，将向外部系统发出请求，然后再对 Foundry 中的对象应用任何更改。
