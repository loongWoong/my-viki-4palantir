# [](#metadata-reference)Metadata reference元数据引用


A link type is represented in the Foundry Ontology by the following metadata:Foundry 本体中，链接类型由以下元数据表示：


- **ID:** A unique identifier of the link type, primarily used to reference links of this type when configuring an application. For example, `employee-employer` may be the ID of the link type defined between the `Employee` and `Company` object types.ID： 链接类型的唯一标识符，主要用于在配置应用时引用此类链接。例如，employee-employer 可能是 Employee 和 Company 对象类型之间定义的链接类型的 ID。
- **RID:** An automatically generated unique identifier for every resource in Foundry. A link type’s RID will be referenced in error messages across the platform.RID： 为 Foundry 中每个资源自动生成的唯一标识符。链路类型的 RID 会在平台的错误信息中被引用。
- **Status:** A signal to users and other Ontology builders about where in the development process the link type stands. It can be `active`, `experimental`, or `deprecated`. By default, the `Employee → Employer` link type will have status `experimental`. Read more about [statuses](/docs/foundry/object-link-types/metadata-statuses/).状态： 这是向用户和其他本体构建者发出的信号，说明链接类型在开发过程中处于何种位置。它可以是主动的、 实验性的，或者已经弃用的。默认情况下， 员工→雇主链接类型将处于实验状态。阅读更多关于状态的信息 。
- **Object types:** The object types related through the link type definition. For example, the `Employee → Employer` link type will refer to the `Employee` and `Company` object types.对象类型： 对象类型通过链接类型定义关联。例如， 员工→雇主链接类型将指向员工和公司对象类型。
- **Cardinality:** Indicates to applications if each object type in the link type has one or many objects. For example, The Employee object type in link type `Employee → Employer` has cardinality `many` and the Company object type has cardinality `one`, since many employees are linked to a single employer. The Employee object types in link type `Direct Report ↔ Manager` will each have cardinality `many` if a direct report can have multiple managers and if a manager can have multiple direct reports.基数： 向应用程序指示链接类型中的每个对象类型是否包含一个或多个对象。例如，员工对象类型在链接 → 类型中具有基数为多 ，而公司对象类型为一 ，因为许多员工关联到同一雇主。链接类型中的员工对象类型  直属下属↔经理中，如果直属下属可以有多个经理，或者如果经理可以有多个直接下属，则每个类型都会有基数。
- **Key:** The properties or columns used to create the links.
说明： 用于创建链接的属性或列。- In a one-to-one or in a one-to-many cardinality link type, a property of one object type (the foreign key) refers to the primary key property of the other object type. This reference between a foreign key and primary key defines the links between objects. For example, in the `Employee → Employer` link type, the `Employee` object type may have an `employer ID` property (the foreign key) that refers to the `company ID` property (primary key) of the `Company` object type.在一对一或一对多基数链路类型中，一个对象类型的属性（外键）指代另一个对象类型的主键属性。外键与主键之间的这种引用定义了对象之间的链接。例如，在员工→雇主链接类型中， 员工对象类型可能包含雇主 ID 属性（外键），该属性指向公司对象类型的公司 ID 属性（主键）。
- In a many-to-many cardinality link type, a table containing pairs of primary keys defines the links between two objects. These link types require a join table to be specified, along with mapping these keys that tell applications which columns in the join table refer to the primary keys of which object types in the link type. For example, the join table backing the `Direct Report ↔ Manager` link type might contain pairs of `employee numbers`, for which each pair represents a `Direct Report ↔ Manager` link.在多对多基数链路类型中，包含主键对的表定义了两个对象之间的链路。这些链接类型需要指定连接表，同时映射这些键，告诉应用程序连接表中哪些列对应链接类型中对象类型的主键。例如，支持直接报告↔管理链接类型的连接表可能包含员工号码对，每对代表一个直接报告↔管理链接。
  - In a one-to-one or in a one-to-many cardinality link type, a property of one object type (the foreign key) refers to the primary key property of the other object type. This reference between a foreign key and primary key defines the links between objects. For example, in the `Employee → Employer` link type, the `Employee` object type may have an `employer ID` property (the foreign key) that refers to the `company ID` property (primary key) of the `Company` object type.在一对一或一对多基数链路类型中，一个对象类型的属性（外键）指代另一个对象类型的主键属性。外键与主键之间的这种引用定义了对象之间的链接。例如，在员工→雇主链接类型中， 员工对象类型可能包含雇主 ID 属性（外键），该属性指向公司对象类型的公司 ID 属性（主键）。
  - In a many-to-many cardinality link type, a table containing pairs of primary keys defines the links between two objects. These link types require a join table to be specified, along with mapping these keys that tell applications which columns in the join table refer to the primary keys of which object types in the link type. For example, the join table backing the `Direct Report ↔ Manager` link type might contain pairs of `employee numbers`, for which each pair represents a `Direct Report ↔ Manager` link.在多对多基数链路类型中，包含主键对的表定义了两个对象之间的链路。这些链接类型需要指定连接表，同时映射这些键，告诉应用程序连接表中哪些列对应链接类型中对象类型的主键。例如，支持直接报告↔管理链接类型的连接表可能包含员工号码对，每对代表一个直接报告↔管理链接。
  
  - **Display name:** The name shown to anyone accessing a link of this type in user applications. Each side of a link type has a display name. A side of a link type represents the link *to* that object type. For example, In the `Employee → Employer` link type, the display name for the `Employee` object type is `Employee` and the display name for the `Company` object type is `Employer`.显示名称： 用户应用中访问此类链接时会显示该名称。链接类型的每一侧都有显示名称。链接类型的一侧代表指向该对象类型的链接。例如，在 Employee → Employer 链接类型中，Employee 对象类型的显示名为 Employee，Company 对象类型的显示名为 Employer。
- **Plural display name:** The name shown to anyone accessing a link of this type with many linked object types in user applications. For example, In the `Employee → Employer` link type, the plural display name for the `Employee` object type is `Employees` and there is no plural display name for the `Company` object type, as there can only be one company per employee.复数显示名称： 用户应用中访问此类链接时显示的名称，该链接包含多个关联对象类型。例如，在 Employee → Employer 链接类型中，Employee 对象类型的复数显示名称为 Employees，Company  对象类型没有复数显示名称，因为每个员工只能有一家公司。
- **API name:** The name used when referring to the link type programmatically in code. The API name on a side of a link type can be used to return objects of that type. For example, if the API name on the Employee side of the `Employee → Employer` link type is `employee`, then calling `Company.employee.get()` will return the `Employee` objects linked to those `Company` objects. Read more about [API names](/docs/foundry/functions/api-objects-links/).API 名称： 在代码中编程指代链接类型的名称。链接类型一侧的 API 名称可以用来返回该类型的对象。例如，如果 Employee → Employer 链接类型中 Employee 端的 API 名称为 employee，那么调用 Company.employee.get（） 会返回与这些公司对象关联的 Employee 对象。阅读更多关于 API 名称的信息。
- **Visibility:** An indication to user applications for how prominently to display the side of the link type (referring to links *to* the object type on that side). A `prominent` side of a link type will lead applications to show this side of the link type first to users. A `hidden`  side of a link type will not appear in user applications. By default, the Employee and Company sides of the link type will have visibilities `normal`.能见度： 为用户应用程序提供指向链接类型侧（指该侧指向对象类型的链接） 应如何显著显示的指示。链接类型的显著一面会促使应用程序首先向用户展示这一侧。链接类型的隐藏面不会出现在用户应用程序中。默认情况下，员工和公司连接类型的可见性通常为正常 。
- **Type classes:** Additional metadata that are interpreted by user applications. Read more about [type classes](/docs/foundry/object-link-types/metadata-typeclasses/).类型类别： 用户应用程序解释的额外元数据。阅读更多关于类型类别的信息 。
