# [](#shared-ontologies)Shared ontologies共享本体论


A shared ontology is a special kind of ontology that allows multiple [organizations](/docs/foundry/security/orgs-and-spaces/#organizations) to collaborate on shared workflows. Shared ontologies live under a dedicated shared [space](/docs/foundry/security/orgs-and-spaces/#spaces) created specifically for the workflow. A shared ontology enables resources from multiple organizations to be used in a workflow, whereas an ontology is private to one organization and contains resources from that organization alone.共享本体是一种特殊的本体，允许多个组织在共享工作流上协作。共享本体存在于专门为工作流程创建的共享空间中。共享本体允许多个组织的资源在工作流程中使用，而本体则是私有的，仅包含该组织的资源。


A shared ontology is automatically created when a new shared space is created. It holds the same organizations [markings](/docs/foundry/security/markings/) and [role grants](/docs/foundry/security/projects-and-roles/#roles) as the shared space.当创建新的共享空间时，共享本体会自动生成。它拥有与共享空间相同的组织标识和角色授权 。


See [cross-organization collaboration](/docs/foundry/security/cross-organization-collaboration/) for more details on how to set up secure collaborative workflows.有关如何建立安全协作工作流程的详细信息，请参见跨组织协作 。
